# Contributing to Support Docs

Thank you for your interest in improving the AI Supreme Council support documentation!

## How to Contribute

### Documentation Improvements

If you find errors, unclear sections, or missing information:

1. **Fork this repository**
2. **Make your changes** to the relevant markdown file
3. **Submit a pull request** with a clear description of your changes

### Translations

We welcome translations of support documentation into other languages. Please:

- Create a new directory named after the language code (e.g., `es/`, `fr/`, `ja/`)
- Translate all markdown files while maintaining structure
- Submit a pull request with language code in the title

### Bug Reports

If you find issues in the documentation:

- Open an issue describing the problem
- Include the file name and section reference
- Suggest the correction if possible

### FAQ Suggestions

Have a question that should be in the FAQ?

- Open an issue with the "FAQ" label
- Include the question and suggested answer
- If it's platform-specific, note the context

## Guidelines

### Style

- Use clear, concise language
- Avoid jargon where possible
- Include code examples in code blocks
- Use proper markdown formatting (headers, lists, links)
- Maintain consistent formatting with existing docs

### Accuracy

- Verify technical information before submitting
- Test steps if they involve the platform
- Update information if it becomes outdated
- Cite sources for third-party information

### Tone

- Be helpful and professional
- Assume the reader may be new to the platform
- Explain why, not just what
- Avoid assumptions about user knowledge

## What We're Looking For

### High Priority

- Corrections to factual errors
- Steps that don't work as documented
- Security-sensitive information that needs updating
- Accessibility improvements

### Medium Priority

- Clarifications of unclear sections
- Additional examples or screenshots
- Better organization of information
- Translation of existing content

### Low Priority

- Minor formatting improvements
- Style preferences
- Opinion-based changes

## Pull Request Process

When submitting a PR:

1. **Check existing issues** - Someone may already be working on it
2. **Keep changes focused** - One PR, one purpose
3. **Write clear descriptions** - Explain what you changed and why
4. **Reference issues** - Link to related issues with `Fixes #123`

## Code of Conduct

Be respectful to all contributors and users:

- Use welcoming and inclusive language
- Accept constructive feedback gracefully
- Focus on what is best for the community
- Assume good intentions

---

## Recognition

Contributors who make significant improvements to the documentation will be recognized in the repository's contributors list and in release notes.
